## English Version

### Description
In this project, you will sort data in a stack using a limited set of instructions, aiming to achieve the lowest possible number of actions. To succeed, you will need to work with various algorithms and choose the most appropriate one for optimized data sorting.
The project involves writing a C program called `push_swap` that calculates and displays the shortest sequence of instructions needed to sort the given integers.

### Mandatory Part
* **Program name**: `push_swap`
* **Files to Submit**: `Makefile`, `*.h`, `*.c`
* **Arguments**: stack `a`: A list of integers
* **External Functions**: `read`, `write`, `malloc`, `free`, `exit`, `ft_printf` (or equivalent)
* **Libft Authorized**: Yes
* **Description**: Sort stacks

**Rules**:
* You have 2 stacks named `a` and `b`.
* Stack `a` contains a random number of unique integers. Stack `b` is empty.
* The goal is to sort numbers in stack `a` in ascending order.
* Operations:
  * `sa`, `sb`, `ss`: Swap the first 2 elements at the top of stack `a`/`b`.
  * `pa`, `pb`: Push the top element from one stack to the other.
  * `ra`, `rb`, `rr`: Rotate all elements of stack `a`/`b` up by 1.
  * `rra`, `rrb`, `rrr`: Reverse rotate all elements of stack `a`/`b` down by 1.

**Benchmark**:
To validate the project with maximum score:
* Sort 100 random numbers in fewer than 700 operations.
* Sort 500 random numbers in fewer than 5500 operations.

### Instructions
**Compilation**
Run `make` to compile the `push_swap` executable.
```bash
make
```

**Usage**
Run the program with a list of integers as arguments:
```bash
./push_swap 2 1 3 6 5 8
```
The program will output the list of operations to sort the stack.

**Error Handling**
The program outputs "Error" followed by a `\n` to stderr if:
* Arguments are not integers.
* Arguments exceed integer limits.
* There are duplicates.
